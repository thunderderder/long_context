═══════════════════════════════════════════════════════
   🚀 三种部署方式，任选其一：
═══════════════════════════════════════════════════════

方式 1️⃣ : Docker 本地部署（最简单，2分钟）
────────────────────────────────────────────
   适用：本地测试、内网使用
   
   Windows:
   1. 双击运行 deploy_docker.bat
   2. 编辑 .env 文件，设置 API Key
   3. 访问 http://localhost
   
   Linux/Mac:
   1. chmod +x deploy_docker.sh
   2. ./deploy_docker.sh
   3. 编辑 .env 文件，设置 API Key
   4. 访问 http://localhost

方式 2️⃣ : 免费云平台（推荐，5-10分钟）
────────────────────────────────────────────
   适用：公网访问、个人项目
   成本：免费（有额度限制）
   
   步骤：
   1. 查看 QUICKDEPLOY.md 详细步骤
   2. 后端部署到 Railway（https://railway.app）
   3. 前端部署到 Vercel（https://vercel.com）
   4. 完成！

方式 3️⃣ : 云服务器/VPS（20-30分钟）
────────────────────────────────────────────
   适用：商业项目、高流量
   成本：$10-20/月
   
   步骤：
   1. 查看 DEPLOYMENT.md 的"方案三"
   2. 购买云服务器（阿里云/腾讯云/AWS）
   3. 安装 Docker
   4. 部署应用
   5. 配置域名和 SSL

═══════════════════════════════════════════════════════
   ⚠️ 重要安全提示
═══════════════════════════════════════════════════════

❌ 不要：
   - 将 API Key 写在代码中
   - 将 .env 文件提交到 Git
   - 在生产环境开启 debug 模式
   - 使用 CORS = "*" 在生产环境

✅ 务必：
   - 使用环境变量存储 API Key
   - 检查 .gitignore 生效
   - 生产环境设置 DEBUG=False
   - 配置正确的 CORS 域名
